SmartLog Scanner App

A .NET MAUI desktop application that reads student QR codes (via camera or USB keyboard wedge scanner) and submits attendance scans to the SmartLog Web Application. Designed for offline-first resilience with automatic background sync.

Target Platforms
  • macOS (Mac Catalyst) — camera + USB
  • Windows — camera + USB
Key Capabilities
  • Dual scanner support (camera & USB)
  • HMAC-SHA256 QR validation
  • Offline scan queuing with SQLite
  • Audio/visual feedback

Camera QR Scanner

CameraQrScannerService
  • Uses ZXing.Net.Maui for real-time camera barcode detection
  • Debounce: 500ms raw payload debounce prevents processing the same QR frame multiple times
  • Supports 1D and 2D barcode formats
  • Platform-specific camera handler on macOS (AVFoundation via CameraQrViewHandler)
  • Fires ScanCompleted event with ScanResult

USB Keyboard Wedge Scanner

UsbQrScannerService
  • USB barcode scanners emulate keyboard input (keystroke injection)
  • Buffer: Accumulates rapid keystrokes (<100ms inter-key) into a payload string
  • Trigger: Enter key or 100ms timeout after last keystroke
  • Prefix filter: Only processes payloads starting with SMARTLOG:
  • Thread-safe buffer access via lock
  • Fire-and-forget async processing after payload assembly

HMAC Validation

QR Payload Format

SMARTLOG:{studentId}:{unixTimestamp}:{base64HmacSha256}

Example: SMARTLOG:2026-07-0001:1739512547:5Dkd9j2xQ8kL...

Validation Steps (HmacValidator)

  1. Split payload on : — must have exactly 4 parts
  2. Verify first part is SMARTLOG (case-sensitive)
  3. Retrieve HMAC secret from SecureStorage via ISecureConfigService
  4. Compute HMAC-SHA256 over {studentId}:{timestamp}
  5. Compare signatures using CryptographicOperations.FixedTimeEquals() (constant-time)
  6. Check QR expiry (default: 2 years from timestamp)
  7. Return HmacValidationResult with StudentId or typed RejectionReason

Rejection Types

RejectionCause
MalformedWrong number of colon-separated parts
InvalidPrefixFirst part is not "SMARTLOG"
InvalidBase64Signature part is not valid Base64
InvalidSignatureHMAC verification failed
SecretUnavailableHMAC secret not configured in SecureStorage
ExpiredQR timestamp older than 2 years
Rejection messages never include the HMAC secret or computed hash values to prevent information leakage.

Scan Deduplication

ScanDeduplicationService

Student-level duplicate detection with tiered time windows. Uses ConcurrentDictionary for thread safety.

WindowActionUI Effect
< 2 secondsSuppressSilently ignored (no feedback)
2–30 secondsWarnAmber feedback + duplicate sound
> 30 secondsProceedNormal processing

Cache TTL: 5 minutes per student entry. Expired entries are cleaned up automatically.

Offline Queue

OfflineQueueService (SQLite Persistence)

When the server is unreachable, scans are persisted to a local SQLite database for later synchronization.

StatusMeaning
PENDINGAwaiting sync to server
SYNCEDSuccessfully submitted, ServerScanId recorded
FAILEDPermanently failed after 10 sync attempts

Key Operations

  • EnqueueScanAsync() — Add scan with PENDING status
  • GetPendingScansAsync(limit) — Fetch in FIFO order
  • MarkSyncedAsync(id, serverScanId) — Mark successful
  • MarkFailedAsync(id) — Increment SyncAttempts, mark FAILED after 10
  • HasPendingForStudentAsync(studentId, scanType) — Prevent duplicate enqueue
  • DeleteSyncedScansAsync() — Clean up records older than 7 days

Background Sync

BackgroundSyncService

Automatically syncs queued scans when the server becomes reachable.

  • Subscribes to IHealthCheckService.ConnectivityChanged event
  • Triggers sync when online status changes to true
  • Batch size: 50 scans per cycle (configurable via OfflineQueue:SyncBatchSize)
  • FIFO ordering: Oldest scans synced first
  • Exponential backoff: 2attempts seconds between retries, capped at 5 minutes
  • Mid-batch failure: Stops immediately if server goes offline during sync
  • Emits SyncCompleted event with counts (synced, failed, skipped)

HTTP Resilience (Polly)

The SmartLogApi named HttpClient has two Polly policies:

Retry Policy
  • 3 retries with exponential backoff
  • Delays: 1s, 2s, 4s
  • Handles transient HTTP errors (5xx, network failures)
Circuit Breaker
  • Opens after 5 consecutive failures
  • Break duration: 30 seconds
  • Half-open: tests with single request
  • Logged at Error/Info levels
The HealthCheck HttpClient intentionally has NO Polly policies to ensure fast, deterministic failure detection.

Setup Wizard

First-Launch Flow

  1. Device Detection — Automatically identifies available camera and USB scanner hardware
  2. Server Configuration — Enter server base URL
  3. Credentials — Enter API key and HMAC secret (obtained from SmartLog Web admin)
  4. Options — Select default scan type (ENTRY/EXIT), self-signed cert toggle
  5. Connection Test — Validates server reachability (GET /api/v1/health/details with X-API-Key)
  6. Save — Stores credentials in SecureStorage, settings in Preferences, navigates to main scan page

Setup can be re-entered later to modify configuration (edit mode).

Secure Storage

macOS (Keychain)

Credentials stored in the macOS Keychain via MAUI SecureStorage. Protected by the user's login keychain password.

Windows (DPAPI)

Credentials encrypted using Windows Data Protection API (DPAPI). Tied to the user account and machine.

DEBUG mode: Falls back to MAUI Preferences if SecureStorage is unavailable. PRODUCTION: Fails fast if SecureStorage is not available.

Security Features

  • Credential encryption: API key and HMAC secret in platform-native encrypted storage (never in config files or Preferences)
  • Constant-time HMAC comparison: CryptographicOperations.FixedTimeEquals() prevents timing attacks
  • Secret protection: Rejection messages and log entries never include the HMAC secret or computed hash values
  • Response sanitization: All server response fields validated and length-limited to prevent XSS/injection
  • StudentId format validation: Alphanumeric + hyphen only (whitelist)
  • Certificate pinning: Self-signed certs require exact SHA-256 thumbprint match
  • QR expiry: QR codes expire after 2 years (configurable)
  • Security migration: SecurityMigrationService handles legacy insecure storage fallback

Certificate Pinning

Custom ServerCertificateCustomValidationCallback on both SmartLogApi and HealthCheck HttpClients.

Validation Logic

  1. Accept valid CA-signed certificates (standard TLS)
  2. If AcceptSelfSignedCerts = false: reject all invalid certificates
  3. If AcceptSelfSignedCerts = true:
    • Require CertificateThumbprint to be configured
    • Verify exact SHA-256 thumbprint match
    • Verify hostname matches certificate DNS name (IP addresses exempt)
Self-signed cert mode is disabled by default. Users explicitly enable it during setup for development/testing environments.
