Deployment Guide

SmartLog Scanner App

Windows deployment for school gate scanning stations

Prerequisites Automated Setup Manual Deployment Releases Configuration Multiple Stations Network Troubleshooting

Prerequisites

  • Windows 10 (build 19041 or later) or Windows 11
  • The published SmartLog Scanner build (from GitHub Releases or built from source)
  • Network connectivity to the SmartLog Web App server
  • SmartLog Web App server URL and a registered device API key
  • HMAC secret from the SmartLog Web App administrator

Option A: Automated Setup (Recommended)

The deploy/ folder contains scripts that handle everything for Windows. Run as Administrator.

Deployment Scripts

ScriptPurpose
Setup-SmartLogScanner.batFull automated installation wizard — checks prereqs, installs MAUI workload if needed, builds or copies pre-built files, creates desktop shortcut, optionally adds to startup
Update-SmartLogScanner.batUpdate existing installation — backs up current install, pulls latest code, rebuilds, relaunches. Supports -SkipBackup and -Branch flags

First-Time Install

1
Get the files Download the latest release ZIP from GitHub Releases, or clone the repository.
2
Run the installer Right-click deploy/Setup-SmartLogScanner.bat → Run as administrator.
3
Follow the wizard The script checks for .NET MAUI workload (or uses pre-built files), installs to %LocalAppData%\SmartLogScanner, and creates a desktop shortcut.
4
Complete the app Setup Wizard Enter server URL, API key, and HMAC secret. Click Test Connection, then save.
The script auto-detects whether it is running from a pre-built release ZIP (skips build) or from source (builds from source). Pre-built is preferred for deployment.

Updates

Right-click deploy/Update-SmartLogScanner.bat → Run as administrator. The script backs up the current install, pulls latest code (or extracts a new release ZIP), rebuilds, and relaunches.

Optional flags (when running the .ps1 directly in PowerShell):

.\deploy\Update-SmartLogScanner.ps1 -SkipBackup   # Skip backup (faster)
.\deploy\Update-SmartLogScanner.ps1 -Branch dev   # Pull from a different branch

Option B: Manual Deployment

1. Build (if deploying from source)

On a machine with the .NET 8 SDK and MAUI workload:

dotnet workload install maui-windows
dotnet publish SmartLog.Scanner -f net8.0-windows10.0.19041.0 -c Release -o ./publish/win-x64

The publish/win-x64/ folder is self-contained — no SDK needed on the target machine.

2. Deploy to Target Machine

Copy publish/win-x64/ to the target PC (e.g., C:\SmartLogScanner\). Run SmartLog.Scanner.exe directly.

3. Complete the Setup Wizard

On first launch, the Setup Wizard opens automatically:

FieldWhere to Get It
Server URLStatic IP of the SmartLog server, e.g., http://192.168.1.10:5050
API KeySmartLog Web App → Admin > Register Device (shown once, sk_live_xxx)
HMAC SecretFrom the SmartLog Web App administrator

Click Test Connection to verify before saving.

Releases via GitHub

  • Production releases are published automatically on v* tags via GitHub Actions
  • Each release includes a ZIP with publish/win-x64/ + deploy/ scripts
  • Download the latest release ZIP, extract, and run Setup-SmartLogScanner.bat as Administrator

Configuration Storage

DataStorage
API KeyWindows DPAPI (encrypted, bound to the Windows user account)
HMAC SecretWindows DPAPI
Server URL, scan modeMAUI Preferences (Windows registry under HKCU)

To reset all settings: delete the MAUI Preferences entry for SmartLogScanner and re-run the Setup Wizard.

Multiple Scanner Stations

Each gate PC is a separate device registration in the SmartLog Web App. Each gets its own API key. All point to the same server URL.

DeviceLocationServer URL
Main Gate ScannerMain Entrancehttp://192.168.1.10:5050
Back Gate ScannerBack Entrancehttp://192.168.1.10:5050

Register each device at Admin > Register Device in the SmartLog Web App.

Network Requirements

  • The scanner PC must be on the same LAN as the SmartLog server
  • Server must have a static IP (scanner needs a fixed address to connect to)

Verify connectivity from the scanner PC:

ping 192.168.1.10
curl http://192.168.1.10:5050/health

Expected: ping replies and Healthy response.

For detailed network configuration, see the SmartLog Web App docs/DEPLOYMENT.md, Section: Network Setup for Scanner Devices.

Troubleshooting

ProblemCheck
Setup Wizard can't connectCorrect server URL and port? Run curl http://{server}/health from this PC.
"Invalid API Key" after setupWas the API key copied completely? Re-register the device and copy the full sk_live_xxx key.
Scanner not reading QR codesCamera permissions granted? USB scanner — try a different USB port.
App crashes on launchCheck %LocalAppData%\SmartLogScanner\logs\ for Serilog log files.
Audio not playingSystem volume up? App has audio permissions?
Scans not reaching serverHealth indicator shows offline? Server running? Firewall open on port 5050?

Uninstall

Delete the installation folder (e.g., C:\SmartLogScanner\) and the desktop shortcut. Credentials stored in DPAPI are automatically cleaned up when the Windows user profile is deleted.
