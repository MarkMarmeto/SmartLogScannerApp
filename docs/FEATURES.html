SmartLog Scanner App

.NET MAUI desktop application for school gate attendance scanning

Setup QR Scanning Feedback Health Monitor Deduplication Offline Queue History Performance
⚙️
Setup & Configuration
  • First-launch Setup Wizard — guided configuration
  • Enter server URL, API key, and HMAC secret
  • Connection test validates server before saving
  • Settings persisted — no wizard on subsequent launches
  • Re-accessible anytime via settings screen
  • Secure storage — DPAPI (Windows) / Keychain (macOS)
  • API key and HMAC secret never stored in config files
📷
QR Code Scanning
ModeDescription
Camera ScanningLive camera feed with real-time QR detection (ZXing.Net.Maui). Cross-platform: AVFoundation on macOS, WinUI camera API on Windows.
USB Barcode ScannerAny USB keyboard-wedge scanner. 100ms input timeout distinguishes scanner from keyboard. Plug and play — no special drivers.
Local HMAC Pre-validationQR payload validated locally before network submission. HMAC-SHA256 constant-time comparison. Invalid QR codes rejected before hitting the server.
🎨
Scan Processing & Feedback

ENTRY / EXIT Mode: Toggle between scan types, persists across app restarts, always visible in the UI.

ColorMeaningAudio Cue
Green Scan accepted — student name, grade, section shown Pleasant beep (400Hz, 200ms)
Amber Duplicate scan — already scanned within dedup window Neutral tone (500Hz, 150ms)
Red Scan rejected — invalid QR, inactive student, etc. Error buzz (200Hz, 300ms)
Blue/Teal Informational (e.g., not a school day) Distinct alert tone

Result screen auto-clears after 3 seconds.

Scan Statistics Footer — Live session counters: total scans, accepted, duplicate, rejected.

🟢
Server Health Monitoring
  • Background health check every 15 seconds (GET /api/v1/health)
  • Visual connection status indicator always visible
  • Scanner stops accepting submissions when server is unreachable

📡
Offline Resilience  Implemented — Currently Disabled
  • SQLite-backed queue for scans that fail to submit
  • Background sync flushes queue when connectivity restored
  • Polly retry policies with exponential backoff
  • Circuit breaker (5 failures → 30s open state)
  • Offline Queue page shows pending items and sync status

Disabled by default ("always-online mode"). Code remains in codebase for future activation.

📜
Scan History
  • Scan Logs page shows history of all scans in the current session
  • Timestamps, student IDs, and result statuses
  • Useful for guard review and incident verification
