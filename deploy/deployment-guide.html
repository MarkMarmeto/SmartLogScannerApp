SmartLog Scanner App

Deployment Guide for Windows — Version 1.0.0

This guide covers installing, configuring, and operating the SmartLog Scanner App on a Windows PC. The Scanner App reads student QR codes at school gates and submits attendance data to the SmartLog Web App server.

Related: The SmartLog Web App must be installed and running on a server before setting up the Scanner App. See the Web App Deployment Guide for server setup instructions.

Clone Repository

Open PowerShell and clone the repository:

git clone https://github.com/MarkMarmeto/SmartLogScannerApp.git C:\SmartLogScannerApp

If Git is not installed, download the repository as a ZIP from GitHub and extract it to C:\SmartLogScannerApp.

Run Setup Script

The automated setup script handles building and installing the app:

1Navigate to the deploy folder
cd C:\SmartLogScannerApp\deploy
2Run the setup script as Administrator
powershell -ExecutionPolicy Bypass -File .\Setup-SmartLogScanner.ps1

Or if a .bat launcher is available:

Right-click Setup-SmartLogScanner.bat > Run as administrator

App Setup Wizard (First Launch)

On first launch, the Scanner App shows a setup wizard instead of the scanning interface. This wizard configures the connection to your SmartLog Web App server.

First Launch | v Setup Wizard |--- Device Detection (automatic) | Detects cameras and USB scanners | |--- Server Configuration | Enter: Server URL, API Key, HMAC Secret | |--- Connection Test (optional) | Validates connectivity to server | |--- Save & Continue Stores credentials securely (Windows DPAPI) Navigates to main scanning interface

Server Configuration

In the setup wizard, enter the following:

Server URL

The full URL of your SmartLog Web App server, including the port number.

http://192.168.2.10:8080
Use the server's static IP address, not localhost. The scanner is a separate machine connecting over the LAN. See the Network Setup section for details.

API Key

The API key for this scanner device. To get one:

  1. Log in to the Web App as Admin or SuperAdmin
  2. Go to Admin > Device Management
  3. Click Register Device
  4. Enter a device name (e.g., "Main Gate Scanner") and location
  5. Copy the generated API key (starts with sk_live_)
Save the API key immediately! It is only shown once. If lost, you must register a new device.

HMAC Secret

The shared secret key used to verify QR code signatures. This must match the key configured in the Web App.

  • Get it from: Admin > Settings > QR Code tab in the Web App
  • Or generate a new one using GenerateHMAC.bat in the Web App's deploy folder
  • Make sure the exact same key is set in both the Web App and Scanner App
Tip: Copy the HMAC key carefully. Even a trailing space will cause HMAC validation failures.

Scan Type

Select the default scan type:

  • ENTRY — Records student arrivals (morning gate)
  • EXIT — Records student departures (afternoon gate)

This can be changed at any time from the main scanning interface.

Scanner Mode

The app automatically detects available scanning hardware:

ModeHardwareHow It Works
CameraWebcam (built-in or USB)Uses the camera to scan QR codes visually via ZXing library
USB ScannerUSB barcode/QR scannerReads keyboard-wedge input from USB QR scanners
Camera + USB FallbackBoth availablePrefers camera, falls back to USB if camera fails

Camera Tips

  • Position the camera so students can hold their QR code 15-30 cm away
  • Ensure adequate lighting — QR codes are harder to read in dim conditions
  • External USB webcams generally have better autofocus than built-in laptop cameras
  • If using multiple cameras, the app auto-selects the first available one

USB QR Scanner Tips

  • The USB scanner must be in keyboard-wedge mode (types characters as keyboard input)
  • Most handheld barcode scanners work out of the box
  • No driver installation is typically needed — they appear as standard USB keyboards

Updating the App

To update the Scanner App to the latest version:

1Pull latest code
cd C:\SmartLogScannerApp
git pull origin main
2Rebuild and publish
dotnet publish SmartLog.Scanner -f net8.0-windows10.0.19041.0 -c Release -o C:\SmartLogScanner --nologo
3Restart the app

Close the running Scanner App and relaunch it from the desktop shortcut or:

C:\SmartLogScanner\SmartLog.Scanner.exe
Note: Your server configuration (API key, HMAC secret, server URL) is preserved across updates — it is stored in Windows secure storage, not in the app folder.

How Scanning Works

Once configured, the scanning flow is:

Student presents QR code | v Scanner reads QR payload SMARTLOG:{studentId}:{timestamp}:{hmac} | v Local HMAC validation Verifies QR signature using shared secret | +---------+---------+ | | Valid Invalid | | v v Submit to server Show error POST /api/v1/scans (red indicator) | +----------+----------+----------+ | | | | ACCEPTED DUPLICATE REJECTED OFFLINE (green) (amber) (red) (blue - queued)

Scan Status Indicators

ColorStatusMeaning
GreenACCEPTEDStudent scanned successfully, attendance recorded
AmberDUPLICATESame student scanned within the deduplication window
RedREJECTEDInvalid QR, inactive student, HMAC failure, or non-school day
BlueQUEUEDServer offline — scan saved locally for sync later

Offline Mode

The Scanner App works even when the server is unreachable:

  • QR codes are validated locally using HMAC (no server needed for basic validation)
  • Scans are saved to a local SQLite database as an offline queue
  • A background sync service monitors server connectivity every 15 seconds
  • When the server comes back online, queued scans are automatically submitted in batches of 50
Health checks: The app pings /api/v1/health on the server every 15 seconds. The connection status indicator in the top bar shows the current state.

Resilience Features

FeatureDetails
Retry Policy3 retries with exponential backoff (1s, 2s, 4s)
Circuit BreakerOpens after 5 consecutive failures, waits 30 seconds before retry

Log Files

  • Daily rolling log files: smartlog-scanner-YYYY-MM-DD.log
  • Retention: 31 days
  • Max file size: 100 MB per log file
  • Format: structured text with timestamps, log level, and source context

To view recent logs:

Get-Content "$env:LOCALAPPDATA\SmartLog.Scanner\logs\*.log" -Tail 50

Resetting the App

To reset the Scanner App and start fresh:

  1. Close the Scanner App
  2. Delete the app data folder: 
    Remove-Item "$env:LOCALAPPDATA\SmartLog.Scanner" -Recurse -Force
  3. Clear secure storage (API key, HMAC secret) from Windows Credential Manager:
    • Open Credential Manager (search in Start menu)
    • Under Windows Credentials, remove entries related to SmartLog
  4. Relaunch the app — the setup wizard will appear again

Network Setup

The Scanner App must be on the same local network (LAN) as the SmartLog Web App server.

Verifying Connectivity

From the scanner PC, test connectivity to the server:

REM Check if the server is reachable
ping 192.168.2.10

REM Check if the web app is responding
curl http://192.168.2.10:8080/api/v1/health

Or open http://192.168.2.10:8080/api/v1/health in a browser. You should see a JSON response.

Network Checklist

  • Server PC has a static IP address (so it doesn't change on reboot)
  • Scanner PC and server PC are on the same subnet (e.g., both 192.168.2.x)
  • Server firewall allows inbound connections on port 8080
  • No VPN or network isolation between the two devices
  • If using Wi-Fi on the scanner, ensure it's on the same network as the server's Ethernet

Multiple Scanner Devices

You can have multiple Scanner App instances (e.g., one at each gate). Each scanner needs:

  • Its own API key (register each device separately in the Web App)
  • The same HMAC secret (shared across all devices)
  • The same server URL
Tip: Name each device descriptively when registering (e.g., "Main Gate Scanner", "Back Gate Scanner") so you can identify them in the scan logs.
