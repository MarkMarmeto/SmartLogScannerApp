## HMAC Validation Details

### How It Works

1. **QR Format:** `SMARTLOG:{studentId}:{timestamp}:{hmacBase64}`
2. **HMAC Input:** `{studentId}:{timestamp}` (no "SMARTLOG:" prefix)
3. **HMAC Algorithm:** SHA-256
4. **HMAC Key:** UTF-8 encoded secret string
5. **HMAC Output:** Base64-encoded bytes

### Example Validation (Student 1)

```
Input:
  studentId = "STU-2026-001"
  timestamp = "1773016071"
  secret = "smartlog-test-secret-2026"

HMAC Calculation:
  message = "STU-2026-001:1773016071"
  key = secret.encode('utf-8')
  signature = HMAC-SHA256(key, message)
  hmacBase64 = base64.encode(signature)

Result:
  hmacBase64 = "5leM1OxocegePjRfgsBjOq24fPeNZi1AYaqnJBx+YgM="

Final QR:
  "SMARTLOG:STU-2026-001:1773016071:5leM1OxocegePjRfgsBjOq24fPeNZi1AYaqnJBx+YgM="
```

## Troubleshooting

### ❌ "InvalidSignature: HMAC verification failed"

**Cause:** HMAC secret in app doesn't match secret used to generate QR codes

**Fix:**
```bash
# Run setup script to configure correct secret
/tmp/setup_test_config.sh

# OR manually set in app settings:
# HMAC Secret: smartlog-test-secret-2026
```

### ❌ "Malformed: expected 4 colon-separated parts"

**Cause:** QR code payload is not in correct format

**Fix:** Use QR codes generated by the script (they're in correct format)

### ❌ "SecretUnavailable: HMAC secret not configured"

**Cause:** App doesn't have HMAC secret configured

**Fix:** Run `/tmp/setup_test_config.sh` or manually configure in app settings

### ⚪ App shows "Connecting..." forever

**Cause:** App is trying to reach server but it's not running

**Fix:** Either:
1. Start mock server: `node mock-server/server.js`
2. Or ignore - HMAC validation works locally without server
