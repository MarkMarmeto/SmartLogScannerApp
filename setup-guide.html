SmartLog Scanner App — Windows Setup Guide

Step-by-step guide to install and configure the SmartLog Scanner App on a Windows machine for QR-based attendance scanning.

Automated Setup Available! For a faster installation, use the PowerShell setup script: 
# Open PowerShell as Administrator, navigate to the project folder, then run:
.\deploy\Setup-SmartLogScanner.ps1
The script automates building, publishing, shortcut creation, and optionally auto-launching on startup.

1. Before You Begin

Before setting up the Scanner App, make sure you have:

  • SmartLog Web App running on a server with a known IP address (e.g., 192.168.1.100:8080)
  • A registered scanner device — get the API key from the Web App admin (Admin → Device Management → Register Device)
  • The HMAC secret key — same key configured on the Web App server (SMARTLOG_HMAC_SECRET_KEY)
You need these 3 values from the Web App administrator:
  1. Server URL — e.g., http://192.168.1.100:8080
  2. API Key — e.g., sk_live_abc123... (from device registration)
  3. HMAC Secret — the shared secret used for QR code signing
Without these, the scanner cannot authenticate with the server or validate QR codes.

2. Prerequisites & Downloads

SoftwareVersionDownload
.NET 8.0 SDK 8.0 or later dotnet.microsoft.com
.NET MAUI Workload (included with SDK) Installed via dotnet workload install maui

Hardware: Windows 10/11 (version 1903+), webcam (for camera scanning mode), or USB barcode scanner (keyboard wedge mode). At least 4 GB RAM recommended.

3. Install Required Tools

1 Install .NET 8.0 SDK

  1. Download the .NET 8.0 SDK installer for Windows x64
  2. Run the installer and follow the prompts
  3. Open Command Prompt or PowerShell and verify:
dotnet --version

You should see a version starting with 8.0.

2 Install the MAUI Workload

Open PowerShell as Administrator and run:

dotnet workload install maui

This installs the .NET MAUI development tools needed to build the app. It may take a few minutes.

Verify the installation:

dotnet workload list

You should see maui or maui-windows in the list.

3 Install Git for Windows

  1. Download and run the Git installer with default options
  2. Verify: git --version

4. Clone the Repository

cd C:\
git clone https://github.com/MarkMarmeto/SmartLogScannerApp.git
cd SmartLogScannerApp

5. Build the Application

1 Build for Windows

dotnet build SmartLog.Scanner -f net8.0-windows10.0.19041.0 -c Release

You should see Build succeeded with 0 errors.

2 Publish (Self-Contained)

For deployment to machines that may not have .NET installed:

dotnet publish SmartLog.Scanner -f net8.0-windows10.0.19041.0 -c Release -o C:\SmartLogScanner

This creates a standalone app at C:\SmartLogScanner.

3 Launch the App

Run the published executable:

C:\SmartLogScanner\SmartLog.Scanner.exe

On first launch, you'll see the Device Setup wizard.

6. First Launch & Device Setup

When the app starts for the first time, the setup wizard guides you through configuration:

1 Device Detection

The app automatically detects available scanning hardware:

  • Camera — detects connected webcams
  • USB Scanner — any keyboard-wedge barcode scanner

The recommended scanner mode is auto-selected based on detected hardware.

2 Server Configuration

Enter the connection details for the SmartLog Web App:

FieldExampleNotes
Server URL http://192.168.1.100:8080 The static IP and port of the server on the school LAN
API Key sk_live_abc123... From device registration in the Web App
HMAC Secret (Base64 string) Must match the server's SMARTLOG_HMAC_SECRET_KEY

3 Options

OptionDefaultDescription
Default Scan Type ENTRY Whether scans default to ENTRY or EXIT mode
Accept Self-Signed Certs Off Enable if the server uses a self-signed HTTPS certificate
Sound Enabled On Audio feedback on scan results

4 Test Connection

Click Test Connection to verify the scanner can reach the server. You should see a green success message. If it fails, check:

  • Server URL is correct (include port number)
  • Server is running and accessible on the LAN
  • Firewall on the server allows incoming connections on the configured port
  • If using HTTPS with self-signed cert, enable "Accept Self-Signed Certs"

5 Save & Start Scanning

Click Save Configuration. The app navigates to the main scanning screen. You're ready to scan student QR codes!

7. Configuration Details

Where Settings Are Stored

Encrypted (SecureStorage / DPAPI)

  • API Key
  • HMAC Secret Key

Protected by Windows Data Protection API, tied to the user account.

Preferences (Plain Text)

  • Server URL
  • Scanner Mode (Camera/USB)
  • Default Scan Type (ENTRY/EXIT)
  • Sound Enabled
  • Device ID & Name
  • Accept Self-Signed Certs

Local Database

The app uses a local SQLite database for offline scan queuing and scan history. It is created automatically on first launch at:

%LOCALAPPDATA%\SmartLog.Scanner\smartlog-scanner.db

No SQL Server or external database is needed.

Log Files

Serilog writes daily rolling logs to:

%LOCALAPPDATA%\SmartLog.Scanner\logs\smartlog-scanner-YYYYMMDD.log

Logs are retained for 31 days, with a 100 MB per file limit.

Editing Configuration Later

To change settings after initial setup, navigate to the Setup page from within the app (gear icon or menu). All fields can be updated and re-tested.

8. Camera Setup

If using a webcam for QR scanning:

  1. Connect a USB webcam or use the built-in laptop camera
  2. When prompted by Windows, allow camera access for the SmartLog Scanner app
  3. The app shows a small camera preview in the bottom-right corner of the main screen
  4. Hold the student's QR code in front of the camera — the app auto-detects and processes it
Tips for reliable camera scanning:
  • Ensure good lighting — avoid glare and shadows on the QR code
  • Hold the QR code steady, 15–30 cm from the camera
  • External USB webcams generally perform better than built-in laptop cameras
  • A 1080p webcam with autofocus is recommended

9. USB Scanner Setup

USB barcode scanners work in keyboard wedge mode — they emulate keystrokes, so no driver installation is needed.

  1. Plug in the USB barcode scanner — Windows installs it as a keyboard device automatically
  2. In the Scanner App setup, select USB as the scanner mode
  3. The app listens for rapid keystroke sequences ending with Enter
  4. Point the scanner at a student QR code — the scan is processed automatically
How it works: USB barcode scanners type out the QR code content very fast (all characters within 100ms) and press Enter. The app detects this rapid keystroke pattern, assembles the payload, and only processes payloads starting with SMARTLOG:.
Important: The Scanner App window must be in focus (active/foreground) to receive keyboard input from the USB scanner. If another window is active, the keystrokes will go there instead.

10. Auto-Start on Boot

To have the Scanner App launch automatically when Windows starts:

Option A: Startup Folder Shortcut

  1. Press Win + R, type shell:startup, press Enter
  2. Right-click in the folder → New → Shortcut
  3. Enter the path: C:\SmartLogScanner\SmartLog.Scanner.exe
  4. Name it "SmartLog Scanner"

Option B: Task Scheduler (More Control)

  1. Open Task Scheduler (taskschd.msc)
  2. Click Create Basic Task
  3. Name: SmartLog Scanner Auto-Start
  4. Trigger: When the computer starts (or When I log on)
  5. Action: Start a program
  6. Program: C:\SmartLogScanner\SmartLog.Scanner.exe
  7. Check "Run with highest privileges" if camera access requires elevation

11. Updating the App

:: 1. Close the Scanner App

:: 2. Pull latest source code
cd C:\SmartLogScannerApp
git pull

:: 3. Rebuild and publish
dotnet publish SmartLog.Scanner -f net8.0-windows10.0.19041.0 -c Release -o C:\SmartLogScanner

:: 4. Relaunch the app
C:\SmartLogScanner\SmartLog.Scanner.exe
Your configuration (server URL, API key, HMAC secret) is preserved across updates — it's stored in Windows Preferences and SecureStorage, not in the app folder.

12. Troubleshooting

Connection Issues

ProblemSolution
Connection test fails Verify server URL includes port (e.g., http://192.168.1.100:8080). Ping the server IP from this machine.
SSL/TLS error Enable "Accept Self-Signed Certs" in setup if the server uses a self-signed certificate.
Shows "Offline" after setup The health check runs every 15 seconds. Wait a moment. If it persists, re-check the server URL and firewall.
API key rejected Re-enter the API key from device registration. Keys are case-sensitive and shown only once.

Scanning Issues

ProblemSolution
Camera not detected Check Windows Settings → Privacy → Camera. Ensure the app has camera permission.
USB scanner not working Make sure the app window is in focus. Try scanning into Notepad first to verify the scanner works.
QR code rejected (red) The HMAC secret doesn't match the server. Re-enter the same key used in SMARTLOG_HMAC_SECRET_KEY on the server.
"Already scanned" message Normal — the same student was scanned within the last 5 seconds. Wait and scan again, or it's a genuine duplicate.
Scanner stops after switching pages This was fixed in a recent update. Pull the latest code and rebuild.

Build Issues

ProblemSolution
NETSDK1147: MAUI workload not installed Run: dotnet workload install maui (as Administrator)
Platform not supported Ensure you're building with the Windows target: -f net8.0-windows10.0.19041.0
Build fails with NuGet errors Run: dotnet restore SmartLog.Scanner first to restore packages
